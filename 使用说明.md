# Python虚拟环境操作指南

## 项目信息

- **项目路径**: `~/etf_dingtou_monitor`
- **主程序文件**: `etf_dingtou_monitor.py`

## 操作步骤

### 1. 进入项目目录

```
cd ~/etf_dingtou_monitor
```

### 2. 创建虚拟环境

```
python3 -m venv .venv
```

### 3. 激活虚拟环境

```
source .venv/bin/activate
```

**激活成功标志**: 命令行提示符前会出现 `(.venv)` 标记

### 4. 升级pip（推荐）

```
python -m pip install --upgrade pip
```

### 5. 安装项目依赖

```
pip install -r requirements.txt
```

**加速安装（推荐）**:

```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 6. 运行测试

```
# 如果使用pytest
pytest

# 或直接运行主程序
python etf_dingtou_monitor.py
```

### 7. 退出虚拟环境

```
deactivate
```

## 注意事项

1. **首次使用**: 每次打开新终端窗口时都需要重新激活虚拟环境
2. **依赖文件**: 确保项目目录下存在 `requirements.txt` 文件
3. **权限问题**: 如果遇到权限问题，可以尝试使用 `sudo` 或检查文件权限
4. **Python版本**: 确认使用的是 Python 3.3+ 版本

## 常见问题解决

### 问题1: 激活脚本被阻止执行（Windows）

```
# 以管理员身份运行PowerShell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 问题2: pip安装速度慢

使用国内镜像源，如清华源、阿里源等

### 问题3: requirements.txt文件不存在

确认文件路径是否正确，或手动创建依赖文件

## 总结

通过以上步骤，您可以成功创建隔离的Python环境，安装项目依赖并运行测试。虚拟环境的使用可以有效避免依赖冲突，是Python项目开发的最佳实践。
